# Phase Clip5: Hybrid Recall Quality

## Goal

Improve highlight clipping quality by giving the downstream GPT/editor path a stronger candidate pool without moving analysis, rendering, composition, or export into iOS.

## Architecture

- Cloud backend still owns video analysis, candidate generation, GPT selection/edit planning, rendering, storage, and policy.
- iOS remains a control surface for upload, review, status, preview, download, share, and revision commands.
- No full videos are sent to GPT by this phase.
- No FFmpeg commands are generated by GPT.
- No iOS AVFoundation analysis, composition, or production rendering was added.
- No secrets, R2 credentials, or full presigned URLs were added or printed.

## Changes

### Hybrid Provider Recall

`HOOPS_DETECTION_PROVIDER=hybrid` now runs native candidate detection even when HoopCut returns external detections. Previously a successful external provider response replaced native detection entirely, which could starve GPT of better native candidates if HoopCut returned one weak or limited clip.

Behavior now:

- `hybrid` mode:
  - collect HoopCut/external clips when configured
  - collect native audio/visual/context windows
  - merge both pools
  - dedupe overlaps above `0.55`
  - prefer complete event context over a thin overlapping provider clip
  - cap final clips at `settings.max_returned_clips`
- `hoopcut` mode:
  - preserves external-only behavior when the provider succeeds
  - falls back to native when the provider is absent or returns no clips, matching previous safety behavior

### Native Recall Fallback

When native scoring does not produce a hysteresis segment, fallback now returns up to `settings.max_returned_clips` instead of only 3 candidates. This keeps ordinary/non-GPT candidate generation recall-oriented while preserving min-duration and candidate scoring guards before GPT/editor selection.

### Free Edit Availability

Free cloud AI editing is already configured for 3 renders/day:

- `PLAN_TIER_POLICY_REGISTRY["free"].maxDailyRenders = 3`
- The editing service test suite asserts the default free policy is 3 renders/day.
- No additional change was needed for the requested free editing chance cap.

## Quality Rules

The hybrid merge quality key prioritizes:

- duration at least 2 seconds
- shot setup/outcome context around `eventCenter`
- non-shot clip duration context
- `shouldAutoKeep`
- combined score
- confidence
- visual score
- motion score

This directly targets bad outcomes like 0.1-second clips, clips that start right before the basket, or overlapping provider clips with high scores but missing setup/outcome context.
