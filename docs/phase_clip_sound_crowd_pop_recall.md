# Phase Clip Sound Crowd Pop Recall

## Goal

Use loud crowd/audio pops as a backend recall signal for likely highlights without letting audio alone become a final basketball decision.

## Change

- Expanded native cloud audio-reaction candidate windows to keep more action before the pop:
  - lead-in increased from 2.0s to 2.75s
  - follow-through increased from 1.4s to 1.5s
- Shifted GPT audio-reaction keyframe sampling deeper before the pop:
  - `reactionLeadIn` now samples about 2.2s before the reaction anchor
  - `reactionBuild` now samples about 1.15s before the reaction anchor
- Kept the existing safety rule: `Crowd Reaction` / audio-pop candidates are recall hints only. GPT must see a real basketball action and visible outcome in sampled frames before keeping or claiming a scoring/defensive outcome.

## Architecture

- Cloud backend still owns audio profiling, candidate generation, GPT sampling, edit planning, and rendering.
- iOS behavior is unchanged.
- No full videos are sent to GPT; only validated candidate metadata and sampled keyframes are used.
- No FFmpeg commands are generated by GPT.
